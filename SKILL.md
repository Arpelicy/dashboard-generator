# Skill: 数据看板生成器 (Dashboard Generator)

## 触发词

数据看板、可视化看板、数据大屏、dashboard、看板生成、数据中间层、评级系统、分层看板

## 描述

根据数据源和业务规则，自动生成完整的数据看板项目。包含数据中间层(API)、前端可视化界面、云端部署。

**核心能力**：
- 自动构建数据中间层（API层承担业务逻辑）
- 生成可交互的前端看板界面
- 支持筛选、排序、分页、搜索
- 支持多周期/多维度数据对比
- 一键部署到Vercel

## 适用场景

1. **业务数据看板**：销售、运营、财务数据可视化
2. **评级分层系统**：客户分级、供应商评估、风险评分
3. **周期对比分析**：同比/环比数据展示
4. **快速原型验证**：有数据但需要快速出可视化界面

## 工作流程

### Phase 1: 需求收集

向用户收集以下信息：

```
1. 【数据源】
   - 数据格式：JSON / CSV / Excel / 数据库
   - 数据位置：文件路径或连接信息
   - 数据量级：大约多少条记录

2. 【业务规则】
   - 核心指标：有哪些字段需要展示
   - 判定规则：如何分类/评级（如：N项指标满足M项为优质）
   - 阈值配置：各指标的达标标准

3. 【展示需求】
   - 看板主题色：如红色、蓝色、绿色
   - 图表类型：趋势图、柱状图、饼图、排行榜
   - 筛选维度：按级别、按时间、按类别等
   - 排序方式：支持哪些字段排序

4. 【部署需求】
   - 部署平台：Vercel（推荐）/ 本地预览
   - 项目名称：用于生成目录和URL
```

### Phase 2: 数据处理

1. 读取并分析数据源结构
2. 生成JSON格式的数据文件（如数据量大则分片）
3. 创建数据中间层API：
   - 数据加载模块
   - 筛选/排序/分页逻辑
   - 统计计算模块
   - 阈值判定模块

### Phase 3: 前端生成

1. 生成HTML看板页面
2. 集成Chart.js图表库
3. 实现交互功能：
   - 周期/维度切换
   - 搜索筛选
   - 排序分页
   - 数据导出

### Phase 4: 部署上线

1. 创建Vercel配置文件
2. 推送到GitHub
3. 部署到Vercel
4. 返回可访问的URL

## 代码模板

### 模板1: 数据中间层API (Express)

文件位置：`templates/api-template.js`

关键模块：
- 数据加载：从JSON文件读取数据
- 筛选逻辑：按级别、搜索词、时间范围
- 排序逻辑：支持多字段排序
- 分页逻辑：计算offset和limit
- 统计逻辑：计算总数、占比、平均值
- 阈值判定：根据规则计算评级

### 模板2: 前端看板页面

文件位置：`templates/dashboard-template.html`

关键模块：
- 顶部统计卡片
- 核心洞察区域
- 数据表格（支持排序、筛选、分页）
- 趋势图表
- 排行榜
- 导出功能

### 模板3: Vercel配置

文件位置：`templates/vercel.json`

## 使用示例

**用户输入**：
> 帮我生成一个供应商评级看板，数据在 data/suppliers.json，有5000条记录。需要根据6项指标（质量评分、交货准时率、价格竞争力、服务响应、合作年限、订单量）来评级，满足5项为A级，满足4项为B级，其他为C级。主题色用蓝色，需要趋势图和排行榜，部署到Vercel。

**Skill执行**：
1. 读取 suppliers.json 分析数据结构
2. 生成API层实现评级逻辑
3. 生成蓝色主题的前端看板
4. 创建Vercel配置
5. 部署并返回URL

---

## 注意事项

1. **数据量限制**：JSON文件建议不超过50MB，超过需考虑分片或数据库方案
2. **Vercel限制**：Serverless Function有10MB代码包限制，大数据需外部存储
3. **安全性**：敏感数据不要硬编码，使用环境变量
4. **跨域问题**：API层已配置CORS，支持跨域访问

---

## 技术栈

- **后端**：Node.js + Express
- **前端**：原生HTML/CSS/JS + Chart.js
- **部署**：Vercel Serverless
- **数据存储**：JSON文件
